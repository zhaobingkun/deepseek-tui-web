  • 相比内建工具或简单的提供方设置，MCP 在这里承担什么角色？

读上游文档时最该盯住的部分

  • 连接模型、服务定义，以及 MCP 在配置面的真实位置。
  • 当工具能力超出纯文本补全后，权限边界和操作者预期应该如何设定。
  • “扩能力”和“保持流程可理解、可控”之间的平衡。
  • MCP 概念如何回连到工具边界、运行时和排错页面。

最常见的误区

  • 把 MCP 当成普通配置项，而不是能力边界。
  • 加了服务器却不说明信任模型和失败预期。
  • 只看接入，不看运行时和排错影响。
