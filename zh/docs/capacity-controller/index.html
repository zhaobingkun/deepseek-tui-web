为什么 容量控制器文档 对 DeepSeek TUI 用户值得单独看

容量控制器文档 这类页面的价值，在于它把仓库里的特定主题重新整理成更容易搜索和决策的入口。

直接答案

这页不是替代上游 capacity_controller.md 原文，而是帮助用户更快判断这份文档在什么时候最相关。

把它当成进入正确文档分支和站内指南分支的入口，会更高效。

为什么值得单独成页

会搜到这类页面的人，通常已经不满足于仓库根目录 README 的概览了。他们要的是更窄、更明确的答案。把文档主题拆出来，既更利于承接搜索，也更容易导回站内其他页面。

先看什么

  • 这份文档真正解决的是哪类问题？
  • 它更偏安装、架构，还是维护者视角？
  • 读完之后你该回到站内哪条主线继续看？
