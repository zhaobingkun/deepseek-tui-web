这份文档真正覆盖什么

容量控制器文档解释的是高级编排如何防止工作量、委派或并发超出操作者的控制面。

这页能直接回答的问题

  • 在多步骤代理工作流里，容量控制到底解决什么问题？
  • 它和协调、子代理、运行时决策质量之间有什么关系？
  • 当容量规则开始限制吞吐或委派时，操作者应该预期看到什么？

读上游文档时最该盯住的部分

  • 它保护的是审查质量，而不只是限制吞吐。
  • 容量控制如何和委派逻辑、协调提示词联动。
  • 什么时候限制是安全特性，什么时候反而成了可用性问题。
  • 哪些相邻文档定义了它背后的运行时和编排假设。

最常见的误区

  • 把容量控制只当成性能旋钮。
  • 忽略吞吐限制背后的操作者审查原因。
  • 改协调逻辑时不核对容量控制假设。
