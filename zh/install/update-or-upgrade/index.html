更新与升级

更新 DeepSeek TUI 之所以容易混乱，往往是因为忘了当前命令到底由谁管理

正确的更新路径，取决于最初是谁把这个命令放进你的 shell。多数升级困惑，都来自混用安装方式或忘了当前生效的是哪一路。

命令

npm install -g deepseek-tui@latest cargo install deepseek-tui-cli --locked --force brew upgrade deepseek-tui

只运行与你当前生效安装路线对应的那一条更新命令，不要把多条路线混在同一台机器上同时推进。

为什么升级容易变乱

很多升级问题其实不是升级命令错了，而是有重复安装、旧 PATH 或中途换生态导致的。

升级前先查什么

  • 当前生效的 `deepseek` 命令到底来自哪个包管理器？
  • 你机器上有没有旧的 npm、cargo 或二进制副本？
  • 升级后 shell 会不会仍然指向旧版本？
