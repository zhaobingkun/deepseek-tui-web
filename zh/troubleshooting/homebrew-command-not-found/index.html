Homebrew 路径问题

如果 DeepSeek TUI 通过 Homebrew 安装后仍然 command not found，通常不是没装上，而是 shell 还没读到 brew 路径

Homebrew 用户很容易默认“安装成功 = 命令立刻可用”，但实际问题通常出在 path、profile 或当前终端会话没有刷新。

直接答案

只要 brew 说安装成功，就先查路径和 shell 刷新，不要先重装。

重复安装往往比修路径更慢，也更容易误导判断。

为什么 Homebrew 这条线值得单独做

通用的 command not found 页面会把 npm、cargo 和系统路径混在一起，但 Homebrew 用户更常见的问题是把 brew 的安装成功误解成所有终端都自动可见。

先检查什么

  • Homebrew 安装目录有没有被当前 shell 导出？
  • 安装后是否新开过终端？
  • 你是否混用了别的安装方式的路径假设？
