MCP 排错

只要不把所有失败都当成 MCP 问题，DeepSeek TUI 的 MCP 排错就会简单很多

很多看起来像 MCP 的问题，其实是 provider、配置或路径基础没稳。先把底层装好，再去排 MCP server，本身就能省掉很多时间。

直接答案

先把 MCP 当成高级层，而不是第一嫌疑对象。基础配置没排干净之前，不要先钻进 server 细节。

否则你很容易把时间花在错误的层上。

为什么 MCP 问题很容易看错

因为 MCP 是建立在可用的安装和配置之上的，一旦底层不稳，表面上就会像是 MCP 自己坏了。

先隔离什么

  • 不带 MCP 时，基础 DeepSeek TUI 能不能正常工作？
  • provider 认证和基础配置是不是已经稳定？
  • MCP server 本身是否真的可达、描述也和你预期一致？
