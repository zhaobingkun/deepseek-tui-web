Provider 设置

DeepSeek TUI 的 provider 设置，决定了你的 API Key、endpoint 和运行预期能不能真正对上

很多人停留在“API Key 填哪”这一步，但 provider 选择本身，往往才是认证和运行行为能否匹配的关键。

直接答案

先确定 provider 路线，再让 API Key 和 endpoint 配合这个选择。

如果 provider 模型都没理顺，后面的配置问题会越来越难排查。

为什么 provider 设置值得单独写一页

Provider 问题不等于“API Key 放哪里”。很多用户真正要解决的是：自己到底在走哪条后端路径、默认 endpoint 还适不适用，以及这个选择会如何影响运行时行为。

先检查什么

  • 你到底准备走哪条 provider 路线？
  • 当前 API Key 是不是属于这条路线？
  • 你是不是还在沿用不适合当前 provider 的默认 endpoint？
