配置文件位置

DeepSeek TUI 的配置文件位置判断错了，往往会让本来正确的设置看起来像完全没生效

很多用户真正的问题不是不会写配置，而是改错了文件、参考了别的平台路径，或者忽略了环境变量对当前配置的覆盖。

直接答案

如果你改完配置完全没反应，先确认你编辑的是当前安装路径实际读取的那份文件。

路径判断错，比配置语法错更常见。

为什么这个问题值得单独做一页

很多用户在 API Key、provider、endpoint 都检查过之后，才意识到真正的问题可能只是配置文件位置判断错了。它不是进阶问题，但很容易被误判成高级 bug。

先查什么

  • 你编辑的文件是不是当前安装方式真正使用的那一份？
  • 是不是照搬了别的平台或别的 shell 路径？
  • 当前环境变量会不会覆盖文件里的设置？
