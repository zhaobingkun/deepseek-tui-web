Skills vs Prompts

DeepSeek TUI 里的 skills vs prompts，本质上是在比较“可复用工作流”和值不值得每次重新写提示词

用户搜这个词，通常不是不知道 prompt 是什么，而是已经感受到重复工作，想知道 skills 是否真的解决了另一类问题。

直接答案

一次性任务更适合 prompts，重复出现的工作流更适合 skills。

差别不是文案风格，而是复用、约束和长期维护成本。

为什么这组对比值得单独写

很多人会把 skill 当成“更长的 prompt”。但实际差异在于：skill 更适合稳定复用同一类流程，而 prompt 更适合一次性描述眼前任务。

该比什么

  • 这个工作流会不会重复出现？
  • 你需要稳定结构，还是只需要当次表达？
  • 下次再做同类任务时，还想不想沿用同一套边界？
